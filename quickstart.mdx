## Overview

This guide will help you create a basic audio recorder with real-time visualization in just a few minutes. You'll learn how to use the `useVoiceVisualizer` hook and `VoiceVisualizer` component to capture and display audio recordings.

## Basic Implementation

<Steps>
  <Step title="Import the required components">
    First, import the hook and component from the library:

    ```typescript
    import { useVoiceVisualizer, VoiceVisualizer } from "react-voice-visualizer";
    ```

    These are the two main exports you'll need for most use cases.
  </Step>

  <Step title="Initialize the recorder controls">
    Call the `useVoiceVisualizer` hook to get the recorder controls:

    ```typescript
    const recorderControls = useVoiceVisualizer();
    ```

    The hook returns an object containing all the state and functions you need to manage audio recording and playback.
  </Step>

  <Step title="Render the VoiceVisualizer component">
    Pass the controls to the `VoiceVisualizer` component:

    ```typescript
    <VoiceVisualizer controls={recorderControls} />
    ```

    That's it! The component includes built-in controls for recording, playback, and visualization.
  </Step>
</Steps>

## Complete Example

Here's a complete working example:

```typescript App.tsx
import { useEffect } from "react";
import { useVoiceVisualizer, VoiceVisualizer } from "react-voice-visualizer";

const App = () => {
  // Initialize the recorder controls using the hook
  const recorderControls = useVoiceVisualizer();
  const { recordedBlob, error } = recorderControls;

  // Handle the recorded audio blob
  useEffect(() => {
    if (!recordedBlob) return;

    console.log(recordedBlob);
    // You can now:
    // - Upload the blob to a server
    // - Convert it to a different format
    // - Store it in local storage
  }, [recordedBlob]);

  // Handle errors
  useEffect(() => {
    if (!error) return;

    console.error(error);
  }, [error]);

  return <VoiceVisualizer controls={recorderControls} />;
};

export default App;
```

## Understanding the Controls

The `useVoiceVisualizer` hook returns a controls object with everything you need:

### Key State Values

- **`recordedBlob`** - The recorded audio as a Blob (can be uploaded or saved)
- **`isRecordingInProgress`** - Whether recording is currently active
- **`duration`** - Total duration of the recorded audio in seconds
- **`error`** - Any error that occurred during recording or playback

### Key Functions

- **`startRecording()`** - Start a new recording
- **`stopRecording()`** - Stop the current recording
- **`togglePauseResume()`** - Pause/resume recording or playback
- **`clearCanvas()`** - Clear the visualization and reset state

<Note>
  The `VoiceVisualizer` component includes built-in UI controls, so you don't need to create buttons manually. However, you can hide the default controls and build your own using the functions from the hook.
</Note>

## Basic Controls Usage

The `VoiceVisualizer` component includes:

- **Record button** - Click to start recording (microphone permission required on first use)
- **Pause/Resume button** - Pause and resume during recording or playback
- **Stop button** - Stop the recording and process the audio
- **Play button** - Play back the recorded audio
- **Clear button** - Remove the recording and start over

The visualization shows:
- Real-time audio waveform during recording
- Interactive waveform after recording (click to seek)
- Progress indicator during playback

## Working with the Recorded Audio

The `recordedBlob` can be used in multiple ways:

### Upload to Server

```typescript
const uploadAudio = async (blob: Blob) => {
  const formData = new FormData();
  formData.append('audio', blob, 'recording.webm');
  
  await fetch('/api/upload', {
    method: 'POST',
    body: formData,
  });
};
```

### Save to Local Storage

```typescript
const saveToLocalStorage = async (blob: Blob) => {
  const reader = new FileReader();
  reader.onloadend = () => {
    const base64 = reader.result as string;
    localStorage.setItem('audioRecording', base64);
  };
  reader.readAsDataURL(blob);
};
```

### Download Locally

```typescript
// Use the built-in function
recorderControls.saveAudioFile();

// Or enable the download button
<VoiceVisualizer
  controls={recorderControls}
  isDownloadAudioButtonShown={true}
/>
```

<Note>
  Recordings are saved in WebM format by default. If you need a different format (MP3, WAV, etc.), you'll need to convert the blob using a library like FFmpeg.js.
</Note>
